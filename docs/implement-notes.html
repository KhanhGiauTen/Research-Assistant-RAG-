Thiết lập luật agent và tài liệu kiến trúc dài hạn

docs agent-rule governance
Thời gian: 2026-07-02 15:06:43 +07:00 Người note: KhanhGiauTen

Bối cảnh

Cần một luật repo áp dụng cho mọi phiên agent để các quyết định, thay đổi và kiến trúc dự án không bị mất ngữ cảnh giữa các phiên làm việc dài hạn phục vụ nghiên cứu.

Quyết định

  • Dùng AGENTS.md ở root làm luật bắt buộc cho agent/Codex session.
  • Bắt buộc phiên mới đọc docs/ARCHITECTURE.md và docs/implement-notes.html.
  • Dùng docs/implement-notes.html làm nhật ký card HTML, tiếng Việt, mới nhất trước.
  • Dùng docs/ARCHITECTURE.md làm bản đồ kiến trúc để giảm nhu cầu quét toàn repo ở đầu phiên.

File đã chạm

  • AGENTS.md: thêm luật đọc docs, ghi note, xử lý conflict, commit/push.
  • docs/ARCHITECTURE.md: ghi lại kiến trúc hiện tại của backend, frontend, RAG flow, endpoint và lệnh chạy/test.
  • docs/implement-notes.html: tạo nhật ký triển khai dạng card có timestamp và người note.

Tác động

Các phiên sau có điểm vào rõ ràng để hiểu dự án và phải cập nhật note/kiến trúc khi đụng mã nguồn. Quy trình này hỗ trợ tái cấu trúc lớn theo từng commit nhỏ mà vẫn giữ được lịch sử quyết định.

Kiểm thử/xác minh

Thay đổi chỉ là tài liệu/quy trình. Cần kiểm tra git diff, whitespace bằng git diff --check, sau đó commit và push lên remote.
